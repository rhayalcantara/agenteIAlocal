CORREGIDO

Cloudflare Tunnel + mini-bridge local

Tu PC y la de Ranger exponen un FastAPI mínimo vía túnel. Cero costo, cero C#, cero ntempurl.

// PLANCómo se ve concretamente

Dos PCs, dos túneles, una mini-API en Python. Bidireccional sin intermediario.

  1. Mini-bridge HTTP (~30 LOC)

    FastAPI con POST /inbox y GET /poll. Cola en memoria o JSON-file. Auth con header X-Bridge-Token. Mismo binario en ambas PCs.

    STACK · python · ~10 min

  2. Cuenta gratis Cloudflare + túnel nombrado

    cloudflared tunnel create claude-local y cloudflared tunnel create claude-ranger. URLs estables tipo claude-local.tudominio.workers.dev — sobreviven reinicio.

    RED · cloudflared · ~15 min las dos

  3. Servicio persistente

    En Windows: nssm o tarea programada al boot. En Mac/Linux servidor Ranger: launchd o systemd. Mata-y-resucita automático.

    PERSISTENCIA · ~5 min

  4. Cliente del otro lado

    Cada Claude usa Bash/WebFetch para POST al inbox del otro y GET a su propio inbox. Polling 5-10s o webhook si quieres push.

    INTEGRACIÓN · 0 cambios en infra existente
B

Plan B — si Cloudflare cobra

no-ip.com sigue free con renovación cada 30 días (un click). Cambia solo la URL de cada túnel; el FastAPI es portable. También funcionan ngrok (plan free 1 endpoint por cuenta) y localtunnel.
